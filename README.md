# NLPチュートリアル: ニュースタイトル生成

このチュートリアルでは、大規模言語モデル（LLM）の基本的な動作原理（自己回帰生成やデコーディング）を理解し、実際にモデルをファインチューニング（SFT）して、ニュース記事の本文から適切なタイトルを生成できるようすることを目指します。

## 🎯 本題: ニュースタイトル生成タスク

`Qwen/Qwen3-0.6B-Base` モデルを使用し、ニュース記事のタイトルを生成するタスクに取り組みます。

### 📋 課題のルール
- **目標**: `data/test.jsonl` に対して なるべく高い**ROUGE-L スコア** を達成する(目標0.35以上)。
- **モデル**: `Qwen/Qwen3-0.6B-Base` (Baseモデルから学習を開始)
- **評価用スクリプト**: `generate_evaluate.py`
  - ⚠️ **重要**: `generate_evaluate.py` および `data/test.jsonl` は評価基準となるため、**原則変更しないでください**（エラーが出たら、適宜修正して大丈夫です）。

#### 💡 ROUGE-Lとは？
ROUGE (Recall-Oriented Understudy for Gisting Evaluation) は、要約やタイトル生成の評価によく使われる指標です。
中でも **ROUGE-L** は、生成文と正解文の間の「最長共通部分列 (Longest Common Subsequence)」に基づいています。単なる単語の一致だけでなく、文章の構造的な類似性を考慮できるのが特徴です。
`rouge_test.py`で色々試してみてください。なお、日本語でROUGEを動かすためにはトークナイザが必要です。今回は`utils/janome_tokenizer.py`にこれを定義しています。

### ✍️ 取り組む内容
1.  **学習用データの準備**: 配布された `data/train.jsonl` および `data/validation.jsonl` を自由に使用（または加工）してください。
2.  **学習プログラムの作成**: モデルをファインチューニングするためのスクリプトを自身で作成・実行してください。
    
    #### 💡 実装のヒント
    *   **ライブラリ**: `trl` の `SFTTrainer` や`transformers`の`Trainer`などを使うと、楽に実行できます(やりたかったら自前実装でも...)。
    *   **ハイパーパラメーターの設定**:
        *   **コンテキスト長**: 最大 1024 トークン程度の設定がおすすめです

    *   **学習手法**:
        *   フルパラメーターチューニングだけでなく、**LoRA (Low-Rank Adaptation)** などの効率的な手法を用いても構いません。
    *   **計算時間と計画**:
        *   サーバー（robin）でフルパラメーターチューニングを行う場合、1エポックあたり **20〜30分** 程度を要します。試行錯誤の時間を考慮して計画的に進めてください。
    *   **禁止事項**:
        *   **評価用データ (`data/test.jsonl`) は、絶対に学習に使用しないでください。** 正当な評価ができなくなります。
3.  **プロンプトの最適化 (`system_prompt.txt`)**: モデルへの指示文（システムプロンプト）を工夫して、より良いタイトルを出力するように調整してみてもいいかもしれません。

### 📊 評価の実行
以下のようにプログラムを実行することでスコアの測定・記録をやってくれます。
`--model`引数で評価したいモデルのパスを指定できます。

```bash
# SFTなどで学習したモデル（例: sft_output）を評価する場合
python generate_evaluate.py --model sft_output

# ベースモデルの設定でそのまま評価する場合
python generate_evaluate.py --model Qwen/Qwen3-0.6B-Base
```

## 💡 Tips

### 📈 学習の管理と可視化 (Comet / WandB)
学習の進捗や損失（Loss）の曲線を可視化・管理するために、**Comet ML** や **Weights & Biases (WandB)** などの実験管理ツールの利用をおすすめします。`Trainer` 等を用いる場合、これらのツールと簡単に連携でき、ブラウザ上で学習過程を確認できるため非常に便利です。

### GPU リソースの確認と指定
本環境は複数人で共有して利用するため、以下の手順で空いているGPUを確認し、**1枚のGPUのみ**を使用するようにしてください。
今回のモデルサイズ（0.6B）であれば、1枚のリソースで十分に学習が可能なはずです。

1. **空き状況の確認**:
   ```bash
   nvidia-smi
   ```
   `Memory-Usage` が低く、利用されていない番号（0, 1, 2...）を確認します。

2. **使用するGPUの制限**:
   プログラム実行時に環境変数 `CUDA_VISIBLE_DEVICES` を指定することで、特定のGPUのみをプログラムに使用させることができます。
   ```bash
   # 例: 0番のGPUのみを使用する場合
   CUDA_VISIBLE_DEVICES=0 python train.py
   ```

### 長時間の学習 (tmux の活用)
接続が切れても学習が止まらないよう、仮想端末の使用を強く推奨します。
```bash
# 新規セッション開始
tmux new -s nlp_tutorial

# (セッション内で学習実行...)

# セッションを抜ける (デタッチ): Ctrl+B -> D
# セッションに戻る (アタッチ): tmux a -t nlp_tutorial
```
