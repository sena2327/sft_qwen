# Experiment: 2026-04-13_01

## Hypothesis

- SFT 前に、ベースモデル Qwen 単体で ROUGE を高められる system prompt を見つける。

## Change (Only One)

- `system_prompt.txt` の単一運用をやめ、複数の system prompt を比較する。

## Fixed Conditions

- Model: `Qwen/Qwen3-0.6B-Base`
- Seed: `42`（固定）
- Decode params: 固定（`temperature` / `max_tokens` など）
- Dataset: `./data/test.jsonl`

## Method

- `system_prompts` ディレクトリを作成する。
- `system_prompts/system_prompt_01.txt` を配置し、以降 `system_prompt_02.txt` ... を追加して比較する。
- Python から `generate_evaluate.py` を外部呼び出しし、各 prompt の評価を自動実行する。
- 実行結果（ROUGE 平均・標準偏差・ログファイル）を収集し、prompt 間の差異を比較する。

## Command (Prototype)

```bash
python generate_evaluate.py --model Qwen/Qwen3-0.6B-Base
```

## Result

- Effective prompt patterns:
  - English prompt
  - `###` separator
  - Ambiguous-word restriction
- Overall effect size: limited（改善は確認できるが大幅ではない）
- ROUGE-L Mean: `0.2300`
- ROUGE-L Std: `0.0954`
- CSV row: `20260414_103915,Qwen/Qwen3-0.6B-Base,0.2300,0.0954`
## Interpretation

- 学習元が英語の資料であることが少し効いているのではないか。曖昧表現を避けるのはこのタスクにあってるっぽい
